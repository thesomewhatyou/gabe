### Safety Features

Gabe includes several safety features for moderation commands:

1. **Permission Checks** â€” Users must have the required Discord permission
2. **Role Hierarchy** â€” Cannot moderate users with Administrator or same-level moderation permissions
3. **Self-Protection** â€” Cannot target yourself or Gabe
4. **Owner Override** â€” Bot owner (set in `.env`) can bypass permission checks

**Example Safety Messages:**

- "I'm not banning a mod/admin. That's above my pay grade."
- "You can't kick yourself! Just leave if you want out."
- "I'm not timing myself out. That's counterproductive!"

## Using Commands

Gabe supports two command formats:

### Slash Commands (Recommended)

```text
/ban @user reason here
/vibecheck @friend
/purge 50
```

### Classic Prefix Commands

The default prefix is `&` (configurable in `.env` as `PREFIX`).

```text
&ban @user reason here
&vibes @friend
&purge 50
```

You can also use command aliases:

```text
&yeet @user
!shush @noisy
!urmom
```

- `/help` â€” List all available commands
